## РАЗДЕЛ 1: ОБЩАЯ ИНФОРМАЦИЯ

### 1.1. Цели и задачи приложения

DriveLog решает одну боль: владелец автомобиля **не помнит и не контролирует**, когда нужно менять масло, фильтры, шины и проходить ТО. Из-за этого случаются дорогие поломки, перепробег по маслу и штрафы за «лысую» резину. Приложение превращает разрозненные записи в блокноте, чеках и памяти в единую структурированную систему.

Ключевые цели:

1. **Централизованный учёт обслуживания.** Хранить в одном месте всю историю работ по каждому автомобилю: что меняли, когда, при каком пробеге, за сколько и где.
2. **Проактивные напоминания.** Автоматически рассчитывать дату/пробег следующей замены и заранее (за настраиваемое число дней) уведомлять пользователя, чтобы он не пропустил ТО.
3. **Контроль расходов.** Показывать, сколько денег уходит на обслуживание авто по месяцам и категориям, считать средний чек и общую стоимость владения.
4. **Управление сезонной резиной.** Отслеживать летние/зимние комплекты, остаток протектора, пробег на каждом комплекте и напоминать о сезонной смене.
5. **Полная автономность.** Работать на 100% офлайн — данные принадлежат пользователю и лежат на устройстве, без обязательной регистрации и без интернета.
6. **Готовый отчёт для продажи авто.** Экспорт полной истории обслуживания в PDF/CSV, который повышает доверие покупателя и цену автомобиля.
7. **Мультиавто.** Вести несколько машин (личная, семейная, рабочая) под одним пользователем с быстрым переключением между ними.

### 1.2. Целевая аудитория

**Основной сегмент — «Заботливый автовладелец»**

| Характеристика | Описание |
|---|---|
| **Возраст** | 25–55 лет |
| **Пол** | Преимущественно мужчины (≈75%), доля женщин растёт |
| **Род занятий** | Наёмные специалисты, ИП, малый бизнес, водители-частники |
| **Авто** | 1–2 машины возрастом 3–15 лет (после гарантии, обслуживают сами или на «своём» сервисе) |
| **Гео** | Россия и СНГ, климат с выраженной сменой сезонов (важна зимняя/летняя резина) |
| **Цифровая зрелость** | Уверенный пользователь смартфона, привык к мобильным приложениям |

**Боли пользователя:**

- «Когда я в последний раз менял масло — не помню, чек давно выбросил».
- «Пропустил ТО, потому что напоминание было только в голове».
- «Не знаю, сколько проехал на зимней резине и пора ли её менять».
- «Сервис говорит, что пора менять ремень ГРМ — а правда ли пора? Истории нет».
- «Продаю машину, а доказать, что ухаживал за ней, нечем».
- «Веду учёт в заметках/Excel — неудобно, нет напоминаний».

**Вторичные сегменты:** автоэнтузиасты-перфекционисты (ведут детальный лог), семьи с 2–3 авто, небольшие автопарки (2–5 машин).

### 1.3. Ключевые преимущества перед конкурентами

1. **Офлайн-first и приватность.** Полноценная работа без интернета и аккаунта. Данные не уходят на чужие серверы по умолчанию — сильный аргумент для аудитории, чувствительной к приватности.
2. **Умный движок интервалов.** Расчёт следующей замены сразу по двум осям — **километры И время** (что наступит раньше), с учётом типа масла и стиля эксплуатации. У многих конкурентов только одна ось.
3. **Глубокий модуль шин.** Не «галочка о смене», а полноценный учёт двух комплектов: пробег на каждом, остаток протектора в мм, тип (шип/липучка), напоминание по дате И по температуре за окном.
4. **Нативная скорость.** FlashList + локальная SQLite дают мгновенный отклик даже на 1000+ записей — без «спиннеров загрузки», типичных для облачных аналогов.
5. **Понятный «индекс здоровья авто».** Один наглядный показатель (0–100%) вместо россыпи цифр — пользователь за секунду понимает, всё ли в порядке.
6. **Экспорт «под продажу».** Красивый PDF-отчёт истории обслуживания — готовый документ для Avito/покупателя.
7. **Бесплатное ядро без рекламы в интерфейсе учёта.** Монетизация через необязательное Pro (облачная синхронизация, безлимит авто), а не через навязчивую рекламу.

### 1.4. Аналоги и конкуренты

| Приложение | Сильные стороны | Слабые места (что исправляем мы) |
|---|---|---|
| **Drivvo** | Популярно в СНГ, учёт расходов и топлива | Перегруженный интерфейс, реклама в бесплатной версии, слабый модуль шин, интервалы только по одной оси |
| **Fuelio** | Удобный учёт заправок и статистика | Фокус на топливе, а не на ТО; нет проработанного учёта шин и фильтров; требует Google-аккаунт для синка |
| **CARFAX Car Care** | Бренд, интеграция с историей VIN | Работает в основном по рынку США, привязка к их базе, требует онлайн, нет полноценного офлайна |
| **myCARFAX / aCar** | Напоминания, история ТО | Устаревший UI (aCar), англоязычность, нет температурных напоминаний по шинам |
| **Заметки / Excel** | Бесплатно, гибко | Нет автоматических напоминаний, нет расчётов, нет структуры, легко потерять данные |

**Вывод по позиционированию.** Большинство конкурентов либо «топливо-центричны» (Drivvo, Fuelio), либо привязаны к американскому рынку и онлайну (CARFAX). DriveLog занимает нишу **«приватный офлайн-трекер ТО и шин для рынка СНГ с умными напоминаниями и красивым экспортом»**.

## РАЗДЕЛ 2: ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1. Онбординг и регистрация

**Первый запуск — 4 экрана онбординга (свайп-карусель):**

1. **Экран 1 «Вся история в одном месте».** Иллюстрация дашборда. Текст: «Масло, фильтры, шины и ТО — больше ничего не забудете». Кнопка «Далее».
2. **Экран 2 «Умные напоминания».** Иллюстрация push-уведомления. Текст: «DriveLog сам посчитает, когда пора на сервис — по пробегу и по времени». Кнопка «Далее».
3. **Экран 3 «Работает без интернета».** Иллюстрация замка/офлайна. Текст: «Все данные хранятся только на вашем телефоне. Никакой обязательной регистрации». Кнопка «Далее».
4. **Экран 4 «Добавьте свой автомобиль».** CTA-экран. Кнопки: «Начать» (→ добавление авто), мелким шрифтом «У меня уже есть аккаунт» (→ вход).

Онбординг показывается **только при первом запуске** (флаг `onboarding_completed` в локальном хранилище). Доступен повторный просмотр из «О приложении».

**Способы входа (все опциональны — приложение работает как «Гость» по умолчанию):**

| Способ | Назначение | Обязателен? |
|---|---|---|
| **Гость (локально)** | Старт без регистрации, все данные на устройстве | Способ по умолчанию |
| **Email + пароль** | Привязка аккаунта для будущей облачной синхронизации/восстановления | Нет |
| **Sign in with Apple** | Быстрый вход на iOS (требование App Store при наличии стороннего входа) | Нет |
| **Google Sign-In** | Быстрый вход + экспорт в Google Drive | Нет |

> **Важно:** регистрация нужна только для облачных функций (бэкап, синхронизация — v2.0). Базовый учёт полностью доступен гостю.

### 2.2. Добавление автомобиля

**Поля формы:**

| Поле | Тип | Обязательное | Пример |
|---|---|---|---|
| Марка | Выбор из справочника + ручной ввод | Да | Toyota |
| Модель | Текст / выбор | Да | Camry |
| Год выпуска | Число (1950–текущий) | Да | 2018 |
| VIN | Текст (17 симв., валидация) | Нет | JTNBE40K... |
| Госномер | Текст (маска) | Нет | А123ВС 777 |
| Текущий пробег | Число (км/мили) | Да | 87 500 |
| Тип двигателя | Выбор (бензин/дизель/гибрид/электро) | Нет | Бензин |
| Объём двигателя | Число (л) | Нет | 2.5 |
| Цвет | Выбор | Нет | Серебристый |

**Фото автомобиля:** до **5 фото** (камера или галерея), один из которых — обложка (показывается в списке авто и на дашборде). Фото хранятся локально в файловой системе, в БД — только путь.

**Мультиавто:** в бесплатной версии — до **3 автомобилей**, в Pro — без ограничений. При попытке добавить 4-й авто (free) показывается экран апселла Pro.

### 2.3. Главный экран (Dashboard)

Дашборд привязан к **выбранному автомобилю** (переключатель в шапке). Содержит карточки:

1. **Карточка «Индекс здоровья авто»** — крупный круговой индикатор 0–100% с цветовой шкалой:
   - 🟢 **80–100%** — всё в порядке;
   - 🟡 **50–79%** — скоро потребуется обслуживание;
   - 🔴 **0–49%** — есть просроченные/срочные работы.
   Индекс рассчитывается как агрегат по всем активным интервалам (масло, фильтры, шины, ТО): чем ближе/просроченнее работа — тем ниже вклад.
2. **Карточка «Ближайшее ТО»** — название работы, сколько осталось (км и/или дней), прогресс-бар.
3. **Карточка «Масло двигателя»** — пробег с последней замены, осталось до следующей, дата последней замены.
4. **Карточка «Пробег»** — текущий пробег + кнопка быстрого обновления пробега.
5. **Карточка «Сезон шин»** — текущий установленный комплект (лето/зима), пробег на нём, индикатор «пора менять» при подходящей температуре/дате.
6. **Блок «Напоминания»** — список на сегодня/неделю (заголовок, дата, тип, цветовая метка срочности). Тап → детали напоминания.

Внизу — плавающая кнопка **«+ Добавить событие»**.

### 2.4. Учёт замены масла и фильтров

**Форма «Добавить замену»:**

| Поле | Тип | Пример |
|---|---|---|
| Тип работы | Выбор: масло двигателя / масло КПП / масло ГУР / фильтр масляный / фильтр воздушный / фильтр салонный / фильтр топливный | Масло двигателя |
| Дата | Дата (по умолчанию — сегодня) | 12.06.2026 |
| Пробег на момент замены | Число | 87 500 |
| Расходник (марка/артикул) | Текст | Mobil 1 5W-30 |
| Тип масла | Выбор: синтетика / полусинтетика / минеральное | Синтетика |
| Объём залитого | Число (л) | 4.3 |
| Стоимость | Число (валюта) | 6 800 ₽ |
| Место сервиса | Текст | «У Михалыча», или СТО, или «сам» |
| Интервал до следующей | Число км + число месяцев | 8 000 км / 12 мес |
| Фото чека | Изображение(я) | — |
| Заметка | Текст | «Поменял заодно пробку поддона» |

**Автоматический расчёт следующей замены.** После сохранения система создаёт `reminder`:
`next_km = current_km + interval_km`, `next_date = service_date + interval_months`.
Срабатывает то событие, которое наступит **раньше**. Интервалы по умолчанию (редактируются):

| Работа | Интервал по умолчанию |
|---|---|
| Масло двигателя (синтетика) | 10 000 км / 12 мес |
| Масло двигателя (полусинтетика) | 8 000 км / 12 мес |
| Масло КПП | 60 000 км / 36 мес |
| Масло ГУР | 80 000 км / 48 мес |
| Фильтр масляный | 10 000 км (с маслом) |
| Фильтр воздушный | 20 000 км / 12 мес |
| Фильтр салонный | 15 000 км / 12 мес |
| Фильтр топливный | 30 000 км / 24 мес |

Поддержка **фото чека** (до 3 на запись), типов масла (3) и типов фильтров (4) — обязательна.

### 2.5. Учёт шин

**Данные комплекта шин (`tires`):**

| Поле | Тип | Пример |
|---|---|---|
| Сезонность | Выбор: лето / зима / всесезон | Зима |
| Тип зимних | Выбор: шип / липучка (только для зимних) | Шип |
| Размер | Текст (маска 205/55 R16) | 205/55 R16 |
| Производитель / модель | Текст | Nokian Hakkapeliitta 10 |
| Дата покупки/установки | Дата | 15.10.2025 |
| Остаток протектора | Число (мм) | 8.5 |
| Заметка | Текст | «Куплены б/у, 1 сезон» |

**Учёт пробега на установке (`tire_installations`):** при каждой установке/снятии комплекта фиксируется пробег. Система считает, **сколько км проехано на конкретном комплекте суммарно** (по всем периодам установки) и за текущий период. При смене комплекта счётчик нового комплекта стартует с нуля для нового периода, а суммарный пробег предыдущего фиксируется.

**Напоминание о смене шин:**
- **По дате** — настраиваемые даты «осенней» и «весенней» переобувки (по умолчанию 15 октября и 15 апреля);
- **По температуре** — опционально: при появлении интернета приложение проверяет прогноз и при устойчивом переходе через +7 °C (осень) / +7 °C (весна) показывает рекомендацию.

**Отметки:** тип «шип/липучка», остаток протектора в мм с предупреждением при остатке **< 4 мм (зима) / < 1.6 мм (лето)**.

### 2.6. Напоминания и уведомления

- **Push-уведомления** через `expo-notifications`. За **N дней до события** (по умолчанию 3, настраивается: 1/3/7/14 дней). Для пробеговых — при достижении порога после обновления пробега.
- **Автоматически создаваемые напоминания:** замена масла (двигатель/КПП/ГУР), все 4 фильтра, сезонная смена шин, плановое ТО.
- **Отложить (snooze):** кнопки «Отложить на 1 день» / «на 1 неделю» прямо в уведомлении и на экране напоминания.
- **Экран всех уведомлений:** два режима просмотра — **список** (сгруппирован по срочности: просрочено / сегодня / неделя / месяц) и **календарь** (месячная сетка с метками-точками по дням). Тап по событию → детали + действия (выполнить, отложить, удалить).

### 2.7. История обслуживания

- **Лента событий** — хронологический список всех работ по авто (FlashList). Каждая запись: иконка типа, название, дата, пробег, стоимость.
- **Фильтрация** по типу работ (масло / фильтры / шины / ТО / прочее), по диапазону дат, по сервису. Поиск по тексту/заметке.
- **Расходы:** статистика по месяцам (столбчатый график), разбивка по категориям (круговая диаграмма), **средний чек**, суммарные траты за период, стоимость на 1 км.
- **Экспорт:** PDF (красивый отчёт «История обслуживания DriveLog» с логотипом, таблицей работ и итогами — для продажи авто) и CSV (для Excel). Экспорт работает офлайн, файл сохраняется/шарится системным share-листом.

## РАЗДЕЛ 3: НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 3.1. Производительность

- **Холодный старт** приложения — не более **2 секунд** до интерактивного дашборда на устройствах уровня iPhone SE (2020) / Android среднего класса.
- **Списки:** прокрутка истории из **1000+ записей** без подтормаживаний за счёт `FlashList` (виртуализация, переиспользование ячеек).
- **Анимации и переходы** — стабильные **60 fps** (использовать `react-native-reanimated`, нативный драйвер).
- **Размер приложения** — ориентир ≤ 60 МБ (без учёта пользовательских фото).
- Все тяжёлые операции (генерация PDF, импорт/экспорт) — вне UI-потока, с индикатором прогресса.

### 3.2. Безопасность и приватность

- **Локальное хранение по умолчанию.** Вся БД и фото — на устройстве. Облачная синхронизация — строго **опциональна** и с явного согласия.
- **Минимизация данных.** Не собираем персональные данные без согласия. Аналитика — обезличенная и отключаемая.
- **Биометрия.** Опциональная разблокировка входа через Face ID / Touch ID / отпечаток (`expo-local-authentication`). При недоступности биометрии — fallback на PIN.
- **Соответствие правилам сторов** по конфиденциальности (App Privacy / Data Safety): честное декларирование, политика конфиденциальности в приложении.
- Чувствительные токены (для облака) — в защищённом хранилище (`expo-secure-store` / Keychain / Keystore).

### 3.3. Офлайн-режим

- Приложение **полностью функционально без интернета**: добавление/редактирование авто, событий, шин, напоминания, статистика, экспорт PDF/CSV.
- **Очередь фоновых задач:** загрузка фото чеков в облако (если включено) и проверка погоды для шин выполняются **в фоне при появлении сети**; до этого данные доступны локально.
- Push-напоминания планируются локально (`scheduleNotificationAsync`) и срабатывают **без интернета**.

### 3.4. Дизайн и UX

- Соблюдение **iOS Human Interface Guidelines** и **Android Material Design** через кроссплатформенные компоненты с платформенной адаптацией (нативные пикеры, навигация, тактильная отдача).
- **Минимальный размер тач-таргета — 48×48 dp**, отступы между интерактивными элементами ≥ 8 dp.
- **Адаптивность:** корректная вёрстка от **iPhone SE (375 pt)** до **iPad** и крупных Android-планшетов; поддержка landscape для планшетов; учёт «safe area» (вырезы, home-индикатор).
- **Доступность (a11y):** поддержка Dynamic Type / масштабирования шрифта, контраст текста ≥ 4.5:1, метки для скринридеров (VoiceOver / TalkBack).

## РАЗДЕЛ 4: ЭКРАНЫ И ПРОТОТИПИРОВАНИЕ

Ниже — карта из 11 экранов. Навигация: **bottom-tabs** (Гараж, Дашборд, История, Напоминания, Ещё) + стековые экраны деталей/форм.

### 4.1. Экран «Список автомобилей» (Гараж) — *Priority: Critical*

- **Данные:** карточки всех авто (обложка, марка+модель, год, госномер, текущий пробег, мини-индекс здоровья). Кнопка «+ Добавить авто».
- **Действия:** выбрать авто (→ Дашборд), свайп/лонгтап → редактировать/удалить, добавить новый авто.
- **Навигация:** → 4.2 (добавить/редактировать), → 4.3 (дашборд выбранного авто).

### 4.2. Экран «Добавить / редактировать автомобиль» — *Priority: Critical*

- **Данные:** форма полей из п. 2.2 (марка, модель, год, VIN, госномер, пробег, двигатель, цвет), блок фото (до 5).
- **Действия:** ввод/выбор полей, съёмка/выбор фото, назначение обложки, сохранить, отменить, удалить (в режиме редактирования).
- **Навигация:** ← Гараж; после сохранения → Дашборд нового/изменённого авто.

### 4.3. Экран «Главный дашборд» — *Priority: Critical*

- **Данные:** карточки из п. 2.3 (индекс здоровья, ближайшее ТО, масло, пробег, сезон шин, напоминания на неделю). Переключатель авто в шапке.
- **Действия:** обновить пробег, тап по карточке (→ соответствующий раздел), «+ Добавить событие», переключить авто.
- **Навигация:** → 4.4 (добавить событие), → 4.5 (история), → 4.6 (напоминания), → 4.8 (детали шин).

### 4.4. Экран «Добавить событие» — *Priority: Critical*

- **Данные:** выбор типа события (масло / фильтр / шины / ТО / прочее), затем соответствующая форма (п. 2.4 / 2.5), интервалы, стоимость, сервис, фото чека.
- **Действия:** выбрать тип, заполнить поля, прикрепить фото, сохранить (→ автосоздание напоминания), отменить.
- **Навигация:** ← Дашборд/История; после сохранения → обновлённый Дашборд или История.

### 4.5. Экран «История обслуживания» — *Priority: High*

- **Данные:** лента всех событий (FlashList), фильтры по типу/дате/сервису, поиск.
- **Действия:** фильтровать, искать, открыть запись (детали/редактирование), экспорт в PDF/CSV.
- **Навигация:** → деталь события, → 4.7 (статистика расходов).

### 4.6. Экран «Напоминания и уведомления» — *Priority: High*

- **Данные:** список (просрочено/сегодня/неделя/месяц) и календарь; для каждого — тип, дата/пробег, срочность.
- **Действия:** отметить выполненным (→ форма события), отложить (1 день / 1 неделя), удалить, переключить список/календарь, создать ручное напоминание.
- **Навигация:** → 4.4 (выполнить = добавить событие), ← Дашборд.

### 4.7. Экран «Статистика расходов» — *Priority: Medium*

- **Данные:** график трат по месяцам, разбивка по категориям (круговая), средний чек, итого за период, стоимость на 1 км.
- **Действия:** выбрать период (месяц/квартал/год/всё), переключить категории, экспорт.
- **Навигация:** ← История.

### 4.8. Экран «Детали шины» — *Priority: High*

- **Данные:** оба комплекта (лето/зима): размер, производитель, сезон/тип (шип/липучка), остаток протектора (мм), пробег на комплекте (за период и суммарно), дата установки. История установок/снятий.
- **Действия:** «Установить этот комплект» (фиксирует пробег, обнуляет период нового, закрывает предыдущий), редактировать комплект, обновить остаток протектора, добавить новый комплект.
- **Навигация:** ← Дашборд; → форма установки.

### 4.9. Экран «Настройки» — *Priority: Medium*

- **Данные:** все группы из п. 2.8 (тема, единицы, язык, уведомления, безопасность, данные/бэкап, интервалы по умолчанию).
- **Действия:** изменить параметры, экспорт/импорт JSON, включить биометрию/бэкап, очистить данные.
- **Навигация:** → 4.10 (о приложении), → экран Pro (апселл).

### 4.10. Экран «О приложении» — *Priority: Low*

- **Данные:** версия, сборка, ссылки (политика конфиденциальности, поддержка, оценить в сторе), повторный просмотр онбординга.
- **Действия:** открыть ссылки, написать в поддержку, посмотреть онбординг, посмотреть лицензии open-source.
- **Навигация:** ← Настройки.

### 4.11. Экран «Деталь / редактирование события» — *Priority: High*

- **Данные:** все поля сохранённой работы, фото чека (просмотр в полноэкранном режиме), связанное напоминание.
- **Действия:** редактировать, удалить, открыть фото чека, поделиться записью.
- **Навигация:** ← История / Дашборд.

> **Сводка приоритетов:** Critical — 4.1–4.4 (ядро MVP); High — 4.5, 4.6, 4.8, 4.11; Medium — 4.7, 4.9; Low — 4.10.

```sql
  part_name       TEXT,
  oil_type        TEXT,
  volume          REAL,

);
CREATE INDEX idx_services_car   ON services(car_id);
CREATE INDEX idx_services_type  ON services(type);
CREATE INDEX idx_services_date  ON services(service_date);

CREATE TABLE tires (

  manufacturer  TEXT,
  purchase_date TEXT,
  tread_depth_mm REAL,

  note          TEXT
);
CREATE INDEX idx_tires_car    ON tires(car_id);
CREATE INDEX idx_tires_active ON tires(is_active);

CREATE TABLE tire_installations (
  id               INTEGER PRIMARY KEY AUTOINCREMENT,
  tire_id          INTEGER NOT NULL REFERENCES tires(id) ON DELETE CASCADE,
  car_id           INTEGER NOT NULL REFERENCES cars(id) ON DELETE CASCADE,
  installed_at     TEXT NOT NULL,
  installed_mileage INTEGER NOT NULL,
  removed_at       TEXT,
  removed_mileage  INTEGER,
  distance_km      INTEGER
);
CREATE INDEX idx_tinst_tire ON tire_installations(tire_id);
CREATE INDEX idx_tinst_car  ON tire_installations(car_id);

CREATE TABLE reminders (
  id                INTEGER PRIMARY KEY AUTOINCREMENT,
  car_id            INTEGER NOT NULL REFERENCES cars(id) ON DELETE CASCADE,
  service_id        INTEGER REFERENCES services(id) ON DELETE SET NULL,
  type              TEXT NOT NULL,
  title             TEXT NOT NULL,
  due_date          TEXT,
  due_mileage       INTEGER,
  notify_days_before INTEGER NOT NULL DEFAULT 3,
  status            TEXT NOT NULL DEFAULT 'active',
  snoozed_until     TEXT,
  notification_id   TEXT,
  created_at        TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_reminders_car    ON reminders(car_id);
CREATE INDEX idx_reminders_status ON reminders(status);
CREATE INDEX idx_reminders_due    ON reminders(due_date);

CREATE TABLE documents (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  car_id      INTEGER REFERENCES cars(id) ON DELETE CASCADE,
  service_id  INTEGER REFERENCES services(id) ON DELETE CASCADE,
  kind        TEXT NOT NULL,
  file_uri    TEXT NOT NULL,
  remote_url  TEXT,
  uploaded    INTEGER NOT NULL DEFAULT 0,
  created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_docs_car     ON documents(car_id);
CREATE INDEX idx_docs_service ON documents(service_id);

CREATE TABLE settings (
  key   TEXT PRIMARY KEY,
  value TEXT
);
```

## РАЗДЕЛ 8: ДИЗАЙН-РЕФЕРЕНСЫ

### 8.1. Минималистичный / Скандинавский — *авто-аналог: Volvo*

Много «воздуха», светлый фон (белый/светло-серый), один акцентный цвет (спокойный синий или зелёный). Тонкие линии, крупная типографика, плоские иконки, мягкие тени. Индекс здоровья — лаконичное кольцо без градиентов. Ощущение спокойствия, надёжности и порядка — как салон Volvo. Подходит широкой аудитории и хорошо читается на солнце.

### 8.2. Спортивный / Дерзкий — *авто-аналог: BMW M*

Тёмная тема по умолчанию, глубокий графитовый/чёрный фон, акценты насыщенно-красным и неоновыми бликами. Карбоновые текстуры в шапках карточек, «приборная» эстетика (тахометр-индикаторы здоровья), угловатые формы, моноширинные цифры пробега. Энергично, по-гоночному, для автоэнтузиастов. Создаёт ощущение, что приложение — часть «кокпита».

### 8.3. Premium / Luxury — *авто-аналог: Mercedes-Benz*

Глубокие градиенты (тёмно-синий → фиолетовый), стекломорфизм (полупрозрачные «матовые» карточки с размытием), плавные пружинные анимации, золотистые/платиновые акценты. Высокая детализация теней и бликов, дорогая типографика. Ощущение статуса и заботы премиум-класса — как интерфейс MBUX. Для аудитории, ценящей внешний лоск.

> **Рекомендация:** взять стиль **8.1 (Скандинавский)** как базовый для максимальной читаемости и доступности, со **светлой/тёмной** темами, и предложить **8.2** как опциональную «спортивную» тёмную тему для энтузиастов.
